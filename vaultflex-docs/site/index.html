    VaultFlex¶

    Chat with your Knowledge using a hybrid RAG pipeline (Vector + Graph). Built for teams that want full control over their data.

    üöÄ Features¶

    • üìÅ Flexible Knowledge Base Scopes ‚Äî create isolated workspaces per domain or project
    • üß† Hybrid Retrieval ‚Äî combines vector search (FAISS) and knowledge graphs (Neo4j)
    • üîç LLM-Augmented Reasoning ‚Äî responds only from your ingested content (no hallucinations)
    • üßæ Document Parsing ‚Äî PDF, DOCX, Markdown, TXT support
    • ü™Ñ Interactive UI ‚Äî powered by Streamlit, clean and customisable
    • üíæ Medallion Data Architecture ‚Äî Bronze ‚Üí Silver ‚Üí Gold data layers

    üìÅ Medallion Data Architecture¶

    VaultFlex follows a structured Medallion data pipeline to process documents:

    • ü•â Bronze: Raw files uploaded by the user are stored.
    • ü•à Silver: Cleaned, chunked, and standardised text representations (JSON).
    • ü•á Gold: Semantic embeddings stored in FAISS, and knowledge graphs built in Neo4j.

    üõ°Ô∏è Smart Upload: Hash-Based Deduplication¶

    VaultFlex automatically prevents duplicate document ingestion:

    • ‚úÖ On upload, each file is hashed (e.g., SHA256).
    • üö´ If the same file (even renamed) was previously uploaded, it's rejected.
    • üíæ This ensures clean, efficient, and consistent data processing.

    Hashes are tracked in a persistent file: HASH_TRACK_FILE, with scope-aware keys like finance/employee_policy.pdf.

    üß† LLM Usage¶

    VaultFlex calls your selected local LLM to: - Refine vague user questions - Synthesise answers from graph triples and chunks - Avoid hallucination by grounding output in source data

    Supports: deepseek-r1, gemma3, or anything via Ollama.

    üèÅ Quickstart¶

# 2. Install dependencies

# 3. Update .env (see env_template)

# 4. Launch the UI
